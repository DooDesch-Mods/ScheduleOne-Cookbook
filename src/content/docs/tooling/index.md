The tools and external references the community relies on to build, test, and debug Schedule I mods.

## In this section

- **[Decompilers & Inspection](/tooling/decompilers-and-inspection/)** - reading game code and assets
  (dnSpy/dnSpyEx, ILSpy, AssetRipper, Il2CppDumper) and inspecting the live game (UnityExplorer and
  friends).
- **[Mod Managers](/tooling/mod-managers/)** - the Schedule I Mod Manager (SIMM) for managing versions,
  branches, MelonLoader and mods.
- **[Branch & Version Loaders](/tooling/branch-and-version-loaders/)** - S1-Loader for keeping dual
  Mono/IL2CPP installs and swapping between them.
- **[CodeArchiver](/tooling/code-archiver/)** - automated tracking of game code changes across versions,
  plus the RefGen reference-assembly generator.
- **[Schedule I MCP Server](/tooling/s1-mcp-server/)** - live agentic debugging that lets an LLM query and
  drive the running game.
- **[External Documentation](/tooling/external-docs/)** - the community modding docs, MelonLoader,
  FishNet, and S1API references.

For the game's own structure and how to track what changes between updates, see
[Core Concepts](/core-concepts/).
